# Sass / SCSS 完整講義

> Sass（Syntactically Awesome Style Sheets）是最成熟、使用最廣的 **CSS 預處理器（CSS Preprocessor）**。

## 1. 什麼是 Sass？為什麼要用

### 痛點：原生 CSS 的限制

原生 CSS 寫大型專案時常遇到：

- **重複值難維護**：主色 `#3b82f6` 散落在幾十個地方，改一次改到手軟。
- **沒有巢狀**：選擇器層層重複，`.card .header .title` 每次都要重打。
- **無法重用邏輯**：同一組陰影、漸層、置中技巧到處複製貼上。
- **難以模組化**：一個 `style.css` 動輒上千行。

### Sass 的解法

Sass 提供：**變數、巢狀、mixin、函式、繼承、模組化、控制流程**。寫起來像程式，維護性大幅提升。

### SCSS vs Sass 語法

Sass 有兩種語法，**現在幾乎都用 SCSS**：

| 特性 | SCSS（`.scss`）| Sass 縮排語法（`.sass`）|
|------|------|------|
| 大括號 `{}` | 有 | 無（靠縮排）|
| 分號 `;` | 有 | 無 |
| 與 CSS 相容 | 完全相容（CSS 就是合法 SCSS）| 不相容 |
| 推薦度 | 推薦 | 較少用 |

```scss
// SCSS 語法（本教材一律使用這種）
.button {
  color: white;
  background: blue;
}
```

> **重點**：任何合法的 CSS 都是合法的 SCSS，所以你可以把現有 `.css` 直接改成 `.scss` 開始漸進導入。

## 2. 安裝與編譯

Sass 本身**不能被瀏覽器直接讀取**，必須先「編譯」成 `.css`。目前官方主推 **Dart Sass**。

### 前置：安裝 Node.js

npm 方式需要先裝 [Node.js](https://nodejs.org/)：

- 選 **LTS（長期支援）版本**，較穩定。
- 建議安裝在系統槽（預設路徑即可）。

### 方法 A：npm（推薦，最通用）

```bash
# 全域安裝
npm install -g sass

# Mac 使用者若遇到權限錯誤，需加 sudo（管理者權限）
sudo npm install -g sass

# 或裝在專案內
npm install --save-dev sass

# 編譯單一檔案：來源 → 輸出
sass input.scss output.css

# 監看模式：存檔就自動重新編譯
sass --watch input.scss:output.css

# 監看整個資料夾：scss 資料夾 → css 資料夾
sass --watch scss:css
```

> 補充：`sass` 也支援「空格分隔」寫法，例如 `sass style.scss style.css --watch`、`sass style.scss css/style.css`，效果與冒號寫法相同。

### 檢查是否安裝成功

```bash
sass --version
```

有正確安裝就會印出 Sass 版本號。

### 常用編譯參數

```bash
# 輸出樣式：expanded（展開，好讀）/ compressed（壓縮，上線用）
sass input.scss output.css --style=compressed

# 不產生 source map
sass input.scss output.css --no-source-map
```

### 方法 B：VS Code 擴充套件（最適合新手）

安裝 **Live Sass Compiler** 擴充套件 → 右下角點「Watch Sass」→ 存檔即自動產生 `.css`。免裝 Node.js。

**自訂輸出設定**：可修改套件設定檔控制輸出格式與路徑。

1. 開啟命令面板（`F1` 或 `Ctrl + Shift + P`）
2. 輸入 `open settings`
3. 選擇 **Open Settings (JSON)**，放入以下設定：

```json
// css 產出設定
"liveSassCompile.settings.formats": [
  {
    "format": "expanded",   // expanded=展開好讀；compressed=壓縮成一行，正式版本用
    "extensionName": ".css",
    "savePath": "/css/"     // 輸出路徑，可自行更改
  }
]
```

### 方法 C：建置工具整合

實務專案中通常交給 **Vite / Webpack** 處理：

```bash
npm install -D sass
```

Vite 只要安裝 `sass`，直接 `import './style.scss'` 就會自動編譯，不需額外設定。

## 3. 變數 Variables

用 `$` 宣告變數，把重複的值集中管理。

```scss
// 定義變數
$primary: #3b82f6;
$radius: 8px;
$font-stack: 'Noto Sans TC', system-ui, sans-serif;
$spacing: 16px;

.button {
  background: $primary;
  border-radius: $radius;
  font-family: $font-stack;
  padding: $spacing;
}
```

編譯後：

```css
.button {
  background: #3b82f6;
  border-radius: 8px;
  font-family: 'Noto Sans TC', system-ui, sans-serif;
  padding: 16px;
}
```

### 變數的資料型態

Sass 變數可以存放 7 種型態的值：

| 型態 | 說明 | 範例 |
|------|------|------|
| 數值 Number | 可帶或不帶單位 | `12`、`100px` |
| 字串 String | 可帶或不帶引號 | `'Noto Sans TC'`、`bold` |
| 顏色 Color | 各種顏色表示法 | `blue`、`#4cb5fc`、`hsl(204, 97%, 64%)` |
| 列表 List | 空格或逗號分隔 | `1px 2px 3px`、`1px, 2px, 3px` |
| 地圖 Map | 鍵值對 | `('primary': blue, 'danger': red)` |
| 布林 Boolean | 真／假 | `true`、`false` |
| 空值 Null | 空 | `null` |

```scss
$font-size-h1: 50px;              // 數值
$font-color:   rgb(255, 81, 0);   // 顏色
$margin:       10px 20px 30px;    // list
$is-open:      true;              // 布林
$empty:        null;              // 空值
$bg-color: (                      // map
  'blue':   #0059ff,
  'yellow': #ffd900,
  'green':  #73ff00,
);
```

> Null 常用在 mixin 參數的預設值：某屬性傳 `null` 時，Sass 會**直接不輸出**該行 CSS（見第 6 章）。

### 變數作用域與 `!default`、`!global`

```scss
$color: red;          // 全域變數

.box {
  $color: blue;       // 區域變數（只在這個區塊有效）
  color: $color;      // blue
}

.text {
  color: $color;      // red（外層的全域值）
}
```

- `!default`：**只有在該變數尚未被定義時才賦值**，常用於可覆寫的設定檔。

```scss
$primary: #3b82f6 !default;  // 若使用者已定義 $primary 就不覆蓋
```

- `!global`：在區域中強制修改全域變數（少用）。

### Sass 變數 vs CSS 自訂屬性（CSS Variables）

| | Sass 變數 `$x` | CSS 變數 `--x` |
|---|---|---|
| 生效時機 | **編譯時**（編譯後就消失）| **執行時**（瀏覽器讀）|
| 能被 JS 讀寫 | ✗ | ✓ |
| 響應 media query / 動態切換 | ✗ | ✓ |

兩者可以搭配使用：用 Sass 管理設計 token，用 CSS 變數處理需要執行時切換的東西（例如深色模式）。

```scss
:root {
  --primary: #{$primary};  // 用 #{} 插值把 Sass 值塞進 CSS 變數
}
```

> `#{ }` 稱為 **插值（Interpolation）**，可把 Sass 的值插入到選擇器、屬性名、字串中。

只要出現在 **url 路徑／class 名稱／屬性名稱** 這三種位置，就一定要用 `#{}` 把變數包起來，否則變數不會被展開：

```scss
// 1) class 名稱
.#{$name} { width: 10px; }

// 2) url 路徑
.box { background-image: url(./images/#{$link}.jpg); }

// 3) 屬性名稱
.attr { #{$prop}: 10px; }
```

## 4. 巢狀 Nesting 與 `&` 父選擇器

巢狀讓你依照 HTML 結構撰寫，減少重複。

```scss
.card {
  padding: 16px;
  background: white;

  .title {
    font-size: 20px;
    font-weight: bold;
  }

  .content {
    color: #666;
  }
}
```

編譯後：

```css
.card { padding: 16px; background: white; }
.card .title { font-size: 20px; font-weight: bold; }
.card .content { color: #666; }
```

### `&` 父選擇器參照

`&` 代表「目前的父選擇器」，最常用來寫 **狀態（hover/focus）** 和 **BEM 命名**。

```scss
.button {
  background: $primary;

  &:hover {           // → .button:hover
    background: darken($primary, 10%);
  }

  &:focus { outline: 2px solid; }

  &--primary {        // → .button--primary（BEM 修飾子）
    background: green;
  }

  &.is-active {       // → .button.is-active
    font-weight: bold;
  }
}
```

`&` 也可以放在後面，表達「被某個父層包住時」：

```scss
.title {
  color: black;

  .dark-mode & {      // → .dark-mode .title
    color: white;
  }
}
```

### 巢狀屬性

同前綴的屬性也能巢狀（較少用）：

```scss
.box {
  font: {
    family: sans-serif;
    size: 16px;
    weight: bold;
  }
}
```

> **巢狀不要超過 3 層**！過深的巢狀會產生冗長、難覆寫、效能差的選擇器，是常見的壞味道。

## 5. Partials 與模組化 `@use` / `@forward`

### Partials

以 **底線開頭** 命名的檔案（如 `_variables.scss`）稱為 partial，它**不會被單獨編譯成 CSS**，只用來被其他檔案引入。

```
scss/
├── _variables.scss   # 變數
├── _mixins.scss      # mixin
├── _buttons.scss     # 按鈕元件
└── main.scss         # 主檔，把上面組合起來
```

### `@use`（現代做法，取代舊的 `@import`）

```scss
// main.scss
@use 'variables';
@use 'mixins';
@use 'buttons';
```

### 命名空間 Namespace

`@use` 引入的檔案**不會把變數倒進全域**，而是掛在一個 **命名空間** 底下，預設就是檔名（去掉底線與副檔名）。也就是說，`@use 'typography';` 之後，你**必須**寫 `typography.$variable` 才拿得到裡面的東西，直接寫 `$variable` 會編譯錯誤。

```scss
// _typography.scss
$font-size: 16px;
$line-height: 1.5;

// main.scss
@use 'typography';

body {
  font-size: typography.$font-size;    // 正確
  line-height: typography.$line-height;
  // font-size: $font-size;            // 錯誤：Undefined variable
}
```

#### 為什麼要這樣用？

**1. 避免命名衝突。** 這是最主要的理由。不同 partial 很容易取到同名變數，`$font-size` 在字體檔和按鈕檔各定義一份是很正常的事。舊的 `@import` 會把全部變數攤平在同一個全域空間，後載入的直接蓋掉前面的，而且**不會有任何警告**——你只會發現樣式莫名其妙跑掉。有了命名空間，兩者是 `typography.$font-size` 和 `button.$font-size`，天生分開，不可能互相覆蓋。

**2. 看得出來源。** 讀到 `colors.$primary` 就知道這個值定義在 `_colors.scss`，不用全專案搜尋 `$primary` 在哪宣告。檔案一多的時候，這件事的價值會愈來愈明顯。

**3. 明確依賴。** 每個檔案只看得到自己 `@use` 進來的模組。A 檔案 `@use 'colors'` 不會讓 B 檔案也自動拿到 colors 的變數——B 想用就得自己寫一行 `@use 'colors'`。依賴關係寫在檔案最上面，一目了然。

**4. 只載入一次。** 同一個模組被 `@use` 十次，Sass 也只會編譯載入一次，不會像 `@import` 那樣重複輸出 CSS。

#### 改別名與省略命名空間

命名空間太長的話，用 `as` 改一個短別名：

```scss
@use 'typography' as t;

body { font-size: t.$font-size; }
```

如果檔名有路徑，預設命名空間是**最後一段檔名**：

```scss
@use 'abstracts/variables';   // 命名空間是 variables，不是 abstracts/variables
```

想完全省略命名空間，用 `as *`，之後可以直接寫 `$font-size`：

```scss
@use 'typography' as *;

body { font-size: $font-size; }   // 不用前綴
```

但 `as *` 等於放棄了上面講的前三個好處：變數又回到全域、看不出來源、而且如果兩個 `as *` 的模組有同名變數，Sass 會直接報錯。**建議只在兩種情況用**：一是專案唯一的變數入口檔（確定不會撞名），二是自己寫的小型工具檔。其他情況保留命名空間，或用 `as` 取短別名。

### `@forward`：轉發（做給別人用的樣式庫）

`@forward` 讓你把多個 partial 集中成一個入口再對外提供。

```scss
// _index.scss（樣式庫入口）
@forward 'variables';
@forward 'mixins';

// 使用端
@use 'index' as lib;
```

###關於舊的 `@import`

老教材常見 `@import 'variables';`。**Sass 官方已宣布逐步淘汰 `@import`**，因為它會造成全域命名衝突、重複載入。**新專案請一律使用 `@use` / `@forward`**。

## 6. Mixin：`@mixin` / `@include`

Mixin 是「可重用的樣式片段」，可以帶參數，是 Sass 最強大的功能之一。

### 基本用法

```scss
@mixin flex-center {
  display: flex;
  justify-content: center;
  align-items: center;
}

.box {
  @include flex-center;   // 引入
}
```

### 帶參數

```scss
@mixin button($bg, $color: white) {   // $color 有預設值
  background: $bg;
  color: $color;
  padding: 8px 16px;
  border-radius: 6px;
}

.btn-primary { @include button(#3b82f6); }
.btn-danger  { @include button(#ef4444, white); }
.btn-warn    { @include button($bg: #f59e0b); }  // 具名參數
```

### 可變參數與 `@content`

```scss
// 響應式斷點 mixin（超實用！）
@mixin respond($breakpoint) {
  @if $breakpoint == mobile {
    @media (max-width: 767px) { @content; }
  } @else if $breakpoint == tablet {
    @media (max-width: 1023px) { @content; }
  }
}

.container {
  width: 1200px;

  @include respond(mobile) {   // @content 會把這個區塊塞進 media query
    width: 100%;
  }
}
```

編譯後：

```css
.container { width: 1200px; }
@media (max-width: 767px) {
  .container { width: 100%; }
}
```

> **Mixin vs 函式**：mixin 產生「一段 CSS 規則」；函式（下面第 8 章）回傳「一個值」。

## 7. 繼承 `@extend` 與 `%placeholder`

`@extend` 讓一個選擇器繼承另一個選擇器的樣式，做法是**共用選擇器**（而非複製屬性）。

```scss
%card-base {          // % 開頭是「佔位選擇器」，本身不會輸出 CSS
  padding: 16px;
  border-radius: 8px;
  box-shadow: 0 2px 8px rgba(0,0,0,.1);
}

.card       { @extend %card-base; background: white; }
.card-dark  { @extend %card-base; background: #333; }
```

編譯後（注意選擇器被合併，樣式不重複）：

```css
.card, .card-dark {
  padding: 16px;
  border-radius: 8px;
  box-shadow: 0 2px 8px rgba(0,0,0,.1);
}
.card { background: white; }
.card-dark { background: #333; }
```

### `@extend` vs `@mixin` 該用哪個？

| | `@extend` / `%placeholder` | `@mixin` |
|---|---|---|
| 產出 | 合併選擇器，CSS 較精簡 | 每次複製一份樣式 |
| 能帶參數 | ✗ | ✓ |
| 能放 media query 內 | 有限制，容易出錯 | ✓ |
| 建議 | 純共用、無參數時 | **大多數情況優先用 mixin** |

> 實務建議：**不確定就用 mixin**。`@extend` 在大型專案容易產生非預期的選擇器爆炸。

## 8. 函式 `@function` 與內建函式

### 自訂函式

函式用 `@function` 定義、`@return` 回傳一個值。

```scss
@function rem($px, $base: 16px) {
  @return math.div($px, $base) * 1rem;
}

.title {
  font-size: rem(24px);   // → 1.5rem
}
```

> 注意：新版 Sass 除法要用 `math.div()`，需先 `@use 'sass:math';`。

### 常用內建模組與函式

新版 Sass 內建函式分在不同模組，需先 `@use`：

```scss
@use 'sass:math';
@use 'sass:color';
@use 'sass:string';
@use 'sass:list';
@use 'sass:map';
```

**顏色相關（最常用）**：

```scss
color.adjust($primary, $lightness: -10%)   // 調整明度
color.scale($primary, $lightness: 20%)     // 按比例調整
color.mix(red, blue, 50%)                  // 混色
rgba($primary, 0.5)                         // 加透明度
```

> 舊寫法 `darken()` / `lighten()` / `mix()` 仍可用，但官方推薦改用 `color.adjust` / `color.scale`。

**數學相關**：

```scss
math.div(100, 3)     // 除法
math.round(4.6)      // 5
math.ceil(4.1)       // 5
math.percentage(0.5) // 50%
math.max(1, 5, 3)    // 5
```

## 9. 運算 Operators

Sass 支援數學運算，方便算佈局尺寸。

```scss
$gap: 16px;

.col {
  width: math.div(100%, 3);          // 三欄等寬
  padding: $gap * 2;                 // 32px
  margin-bottom: $gap / 2;           // 需用 math.div，見下
}
```

| 運算子 | 說明 |
|--------|------|
| `+ - * /` | 加減乘除 |
| `%` | 取餘數 |
| `== !=` | 相等 / 不相等 |
| `< > <= >=` | 比較 |
| `and or not` | 邏輯 |

> **除法陷阱**：`/` 在 CSS 有特殊意義（如 `font: 16px/1.5`），Sass 新版已**不建議**用 `/` 做除法，請改用 `math.div(a, b)`。

字串也能用 `+` 串接：

```scss
$name: 'icon';
.#{$name}-home { content: 'home'; }   // 搭配插值產生 .icon-home
```

## 10. 控制流程 `@if` / `@each` / `@for` / `@while`

Sass 有像程式語言一樣的流程控制，常用來**自動產生大量重複樣式**。

### `@if` / `@else`

```scss
@use 'sass:color';

@mixin text-color($bg) {
  // 取得 HSL 的明度通道（新版 Dart Sass 寫法）
  @if color.channel($bg, 'lightness', $space: hsl) > 50% {
    color: black;      // 淺底配黑字
  } @else {
    color: white;      // 深底配白字
  }
}
```

### `@each`：遍歷 List / Map

```scss
// 遍歷 List
$sizes: sm, md, lg;
@each $size in $sizes {
  .icon-#{$size} { /* ... */ }
}

// 遍歷 Map（產生一整組顏色工具類）
$colors: (
  'primary': #3b82f6,
  'success': #22c55e,
  'danger':  #ef4444,
);

@each $name, $color in $colors {
  .text-#{$name} { color: $color; }
  .bg-#{$name}   { background: $color; }
}
```

編譯後會自動產生 `.text-primary`、`.bg-primary`、`.text-success`… 一整組。

#### 實用範例：跨瀏覽器前綴

用 `@each` 遍歷各家瀏覽器前綴，一次補齊 `-webkit-`、`-moz-` 等：

```scss
@mixin prefix($property, $value) {
  @each $prefix in '', -webkit-, -moz-, -ms-, -o- {
    #{$prefix}#{$property}: $value;
  }
}

.box {
  @include prefix(transform, translateX(10px));
}
```

#### 實用範例：斷點 map + respond mixin

把斷點集中在一份 map，再用 `@each` 搭配 `@if` 做出響應式 mixin：

```scss
$breakpoints: (
  small:   480px,
  medium:  720px,
  desktop: 988px,
  large:   1366px,
);

@mixin respond($name) {
  @each $viewpoint, $width in $breakpoints {
    @if $name == $viewpoint {
      @media all and (min-width: $width) { @content; }
    }
  }
}

.container {
  @include respond(medium) {
    width: 100%;
  }
}
```

### `@for`：計數迴圈

```scss
// 產生 12 欄網格
@for $i from 1 through 12 {
  .col-#{$i} {
    width: math.percentage(math.div($i, 12));
  }
}
```

- `from 1 through 12`：包含 12（1~12）
- `from 1 to 12`：不含 12（1~11）

### `@while`

```scss
$i: 6;
@while $i > 0 {
  .mt-#{$i} { margin-top: #{$i * 4}px; }
  $i: $i - 1;
}
```

## 11. Maps 與 Lists

### Lists（清單）

以空格或逗號分隔的值集合。

```scss
$fonts: 'Noto Sans TC', system-ui, sans-serif;
$sizes: 8px 16px 24px 32px;

list.nth($sizes, 2)      // 取第 2 個 → 16px（索引從 1 開始）
list.length($sizes)      // 4
```

### Maps（鍵值對，超重要）

Map 是管理**設計系統 token** 的核心工具。

```scss
@use 'sass:map';

$theme: (
  'primary':   #3b82f6,
  'secondary': #64748b,
  'radius':    8px,
);

.button {
  background: map.get($theme, 'primary');   // 取值
  border-radius: map.get($theme, 'radius');
}

map.has-key($theme, 'primary')   // true
map.keys($theme)                 // primary, secondary, radius
```

搭配 `@each` 就能從一份 map 自動生成整套樣式，是 Sass 模組化的精髓。

## 12. 實戰整合：元件化的思維

把前面所有概念組起來，做一個可維護的按鈕系統：

```scss
@use 'sass:map';
@use 'sass:color';

// 1) 設計 token（Map）
$buttons: (
  'primary': #3b82f6,
  'success': #22c55e,
  'danger':  #ef4444,
);

// 2) 共用基礎（mixin）
@mixin button-base {
  display: inline-flex;
  align-items: center;
  padding: 8px 16px;
  border: none;
  border-radius: 6px;
  cursor: pointer;
  font-weight: 600;
  transition: background .2s;
}

// 3) 用迴圈生成各種顏色變體
.btn {
  @include button-base;

  @each $name, $color in $buttons {
    &--#{$name} {
      background: $color;
      color: white;

      &:hover {
        background: color.adjust($color, $lightness: -8%);
      }
    }
  }
}
```

這段會產生 `.btn--primary`、`.btn--success`、`.btn--danger`，每個都有對應 hover。**改 map 就能一次調整整套設計**——這就是 Sass 的價值。

## 13. 如何拆 Sass 架構

前面學的是語法，這一節談的是**檔案怎麼擺**。一個 `style.scss` 寫到兩千行，就算語法全用對了一樣難維護。

參考範例：[sexfat/sassProject](https://github.com/sexfat/sassProject/tree/master/sass)

### 核心原則：一個檔案只負責一件事

拆檔的判斷標準不是「行數多寡」，而是**改動的理由**。改主色會動到哪個檔案？改按鈕圓角會動到哪個檔案？如果答案是同一個，就該拆開。反過來說，如果一個東西永遠會一起改，硬拆成三個檔案只是增加跳檔案的成本。

### 一個實用的目錄結構

```
sass/
├── _var.scss              # 全域變數：色票、字級、斷點
├── base/                  # 全站基礎樣式
│   ├── _setting.scss      # reset、box-sizing、全域預設
│   ├── _color.scss        # 顏色定義
│   └── _font.scss         # 字體與字級
├── mixin/
│   └── _mixin.scss        # 共用 mixin：RWD 斷點、置中、圓角
├── component/             # 可重複使用的元件（不綁位置）
│   ├── _btn.scss
│   ├── _card.scss
│   └── _title.scss
├── layout/                # 版面區塊（綁位置，每頁都有）
│   ├── _header.scss
│   ├── _nav.scss
│   ├── _footer.scss
│   └── _banner.scss
├── page/                  # 單一頁面專屬樣式
│   ├── _index.scss
│   └── _product.scss
├── plugin/                # 第三方套件的樣式覆寫
│   └── _swiper.scss
└── style.scss             # 唯一入口，只做組裝
```

這是業界常見的 **7-1 pattern** 的簡化版。不用一開始就把七個資料夾全建好，**從 `base` / `component` / `layout` 三個開始就夠了**，專案長大再拆。

### 各層的分界線

最容易搞混的是 `component`、`layout`、`page` 這三層，判斷方式：

| 資料夾 | 判斷問題 | 例子 |
| --- | --- | --- |
| `component` | 這東西**搬到別頁還能用**嗎？ | 按鈕、卡片、標籤、表單欄位 |
| `layout` | 這是**版面骨架**、每頁都出現嗎？ | header、nav、footer、sidebar |
| `page` | 這是**某一頁獨有**、別頁不會用嗎？ | 首頁的 hero 動畫、產品頁的比較表 |

一個判斷技巧：**component 不該決定自己的位置**。按鈕元件負責長什麼樣（顏色、圓角、padding），但「靠右對齊、距離上方 20px」該由用它的 layout 或 page 決定。元件一旦寫死 `margin-left: 40px`，換個地方用就爆掉了。

### 入口檔只做組裝

`style.scss` 是唯一會被編譯成 CSS 的檔案，它裡面**不該有任何樣式**，只負責決定載入順序：

```scss
// style.scss —— 唯一入口，不寫樣式
// 1. 抽象層：變數與 mixin（不產生 CSS）
@use 'var';
@use 'mixin/mixin';

// 2. 基礎層
@use 'base/setting';
@use 'base/color';
@use 'base/font';

// 3. 元件層
@use 'component/btn';
@use 'component/card';
@use 'component/title';

// 4. 版面層
@use 'layout/header';
@use 'layout/nav';
@use 'layout/footer';

// 5. 頁面層
@use 'page/index';
@use 'page/product';

// 6. 第三方覆寫（放最後，權重才蓋得過）
@use 'plugin/swiper';
```

**順序是有意義的**：由通用到專屬，由低權重到高權重。base 先出、plugin 覆寫最後出，這樣後面的規則才蓋得過前面的，不需要靠 `!important` 硬幹。

#### 這裡沒寫命名空間，不會報錯嗎？

不會。`@use 'component/btn';` **還是有命名空間**（就是 `btn`，路徑最後一段），只是入口檔引入它的目的是要它**吐出 CSS**，不是要拿它的變數，所以那個命名空間從頭到尾沒被用到——這完全合法。命名空間只有在你**引用成員**時才必須寫出來。

但有一個坑要注意：**命名空間只取路徑最後一段，不同資料夾同檔名會直接撞車**。

```scss
@use 'base/color';
@use 'component/color';
// Error: There's already a module with namespace "color".
```

解法是其中一個用 `as` 改名：

```scss
@use 'base/color';
@use 'component/color' as c-color;
```

同理，`@use 'page/index'` 的命名空間是 `index` 而不是 `page`。這也是為什麼**檔名最好帶上語意**（`_btn.scss`、`_card.scss`）而不是每個資料夾都放一個 `_color.scss`、`_style.scss`。

編譯只要指向入口檔：

```bash
sass sass/style.scss css/style.css --watch
```

其他檔案都是 partial（底線開頭），不會單獨編譯出 CSS。

### 用 `@forward` 做抽象層入口

上面每個 partial 只要用到變數，都得自己寫一長串 `@use`：

```scss
// component/_btn.scss
@use '../var';
@use '../mixin/mixin';
@use '../base/color';
```

檔案一多會很煩。做法是用 `@forward` 把抽象層集中成一個入口：

```scss
// abstracts/_index.scss
@forward 'var';
@forward 'mixin';
@forward 'color';
```

之後每個元件只要一行：

```scss
// component/_btn.scss
@use '../abstracts' as a;   // 資料夾內有 _index.scss 可省略檔名

.btn {
  padding: a.$space-md;
  background: a.$primary;
  @include a.mobile { width: 100%; }
}
```

### 常見錯誤

**1. 抽象層寫出 CSS。** `_var.scss`、`_mixin.scss` 應該只有變數與 mixin 定義，**不該有任何選擇器**。參考 repo 裡的 `_mixin.scss` 混了 `.box`、`.card` 這些實際樣式，那是教學示範方便，實務上要拆出去——否則每個 `@use` 它的檔案都會被塞進一堆不相干的 CSS。

**2. 用 `@import` 而不是 `@use`。** 參考 repo 的 `style.scss` 用的是 `@import`，那是舊寫法（詳見第 5 節）。**新專案一律用 `@use` / `@forward`**，差別在於 `@import` 會把所有變數攤平到全域、同名變數靜默互相覆蓋，`@use` 有命名空間隔離。

**3. 資料夾建太多。** 三個檔案的專案不需要七層目錄。結構是為了解決問題而存在，沒有問題就不要先造結構。

**4. page 層肥大。** 如果 `_index.scss` 長到三百行，代表裡面有東西該被抽成 component。頁面層理想上只放「這頁獨有的排列與微調」。

## 重點速記

- 用 `$` 定義變數集中管理設計值。
- 巢狀不超過 3 層，善用 `&` 寫狀態與 BEM。
- **模組化用 `@use` / `@forward`，別再用 `@import`**。
- 重用邏輯優先用 **mixin**（可帶參數）；純共用無參數才考慮 `@extend`。
- 除法用 `math.div()`，顏色用 `color.adjust()`。
- Map + `@each` = 自動生成設計系統。
- 架構分層 `base` / `component` / `layout` / `page`，入口檔只組裝不寫樣式。
