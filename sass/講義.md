# Sass / SCSS 完整講義

> Sass（Syntactically Awesome Style Sheets）是最成熟、使用最廣的 **CSS 預處理器（CSS Preprocessor）**。
> 它讓你用「變數、巢狀、函式、模組化」等程式語言的能力來寫 CSS，最後編譯成瀏覽器看得懂的純 CSS。

## 1. 什麼是 Sass？為什麼要用

### 痛點：原生 CSS 的限制

原生 CSS 寫大型專案時常遇到：

- **重複值難維護**：主色 `#3b82f6` 散落在幾十個地方，改一次改到手軟。
- **沒有巢狀**：選擇器層層重複，`.card .header .title` 每次都要重打。
- **無法重用邏輯**：同一組陰影、漸層、置中技巧到處複製貼上。
- **難以模組化**：一個 `style.css` 動輒上千行。

### Sass 的解法

Sass 提供：**變數、巢狀、mixin、函式、繼承、模組化、控制流程**。寫起來像程式，維護性大幅提升。

### SCSS vs Sass 語法

Sass 有兩種語法，**現在幾乎都用 SCSS**：

| 特性 | SCSS（`.scss`）| Sass 縮排語法（`.sass`）|
|------|------|------|
| 大括號 `{}` | 有 | 無（靠縮排）|
| 分號 `;` | 有 | 無 |
| 與 CSS 相容 | 完全相容（CSS 就是合法 SCSS）| 不相容 |
| 推薦度 | 推薦 | 較少用 |

```scss
// SCSS 語法（本教材一律使用這種）
.button {
  color: white;
  background: blue;
}
```

> **重點**：任何合法的 CSS 都是合法的 SCSS，所以你可以把現有 `.css` 直接改成 `.scss` 開始漸進導入。

## 2. 安裝與編譯

Sass 本身**不能被瀏覽器直接讀取**，必須先「編譯」成 `.css`。目前官方主推 **Dart Sass**。

### 前置：安裝 Node.js

npm 方式需要先裝 [Node.js](https://nodejs.org/)：

- 選 **LTS（長期支援）版本**，較穩定。
- 建議安裝在系統槽（預設路徑即可）。

### 方法 A：npm（推薦，最通用）

```bash
# 全域安裝
npm install -g sass

# Mac 使用者若遇到權限錯誤，需加 sudo（管理者權限）
sudo npm install -g sass

# 或裝在專案內
npm install --save-dev sass

# 編譯單一檔案：來源 → 輸出
sass input.scss output.css

# 監看模式：存檔就自動重新編譯
sass --watch input.scss:output.css

# 監看整個資料夾：scss 資料夾 → css 資料夾
sass --watch scss:css
```

> 補充：`sass` 也支援「空格分隔」寫法，例如 `sass style.scss style.css --watch`、`sass style.scss css/style.css`，效果與冒號寫法相同。

### 檢查是否安裝成功

```bash
sass --version
```

有正確安裝就會印出 Sass 版本號。

### 常用編譯參數

```bash
# 輸出樣式：expanded（展開，好讀）/ compressed（壓縮，上線用）
sass input.scss output.css --style=compressed

# 不產生 source map
sass input.scss output.css --no-source-map
```

### 方法 B：VS Code 擴充套件（最適合新手）

安裝 **Live Sass Compiler** 擴充套件 → 右下角點「Watch Sass」→ 存檔即自動產生 `.css`。免裝 Node.js。

**自訂輸出設定**：可修改套件設定檔控制輸出格式與路徑。

1. 開啟命令面板（`F1` 或 `Ctrl + Shift + P`）
2. 輸入 `open settings`
3. 選擇 **Open Settings (JSON)**，放入以下設定：

```json
// css 產出設定
"liveSassCompile.settings.formats": [
  {
    "format": "expanded",   // expanded=展開好讀；compressed=壓縮成一行，正式版本用
    "extensionName": ".css",
    "savePath": "/css/"     // 輸出路徑，可自行更改
  }
]
```

### 方法 C：建置工具整合

實務專案中通常交給 **Vite / Webpack** 處理：

```bash
npm install -D sass
```

Vite 只要安裝 `sass`，直接 `import './style.scss'` 就會自動編譯，不需額外設定。

## 3. 變數 Variables

用 `$` 宣告變數，把重複的值集中管理。

```scss
// 定義變數
$primary: #3b82f6;
$radius: 8px;
$font-stack: 'Noto Sans TC', system-ui, sans-serif;
$spacing: 16px;

.button {
  background: $primary;
  border-radius: $radius;
  font-family: $font-stack;
  padding: $spacing;
}
```

編譯後：

```css
.button {
  background: #3b82f6;
  border-radius: 8px;
  font-family: 'Noto Sans TC', system-ui, sans-serif;
  padding: 16px;
}
```

### 變數的資料型態

Sass 變數可以存放 7 種型態的值：

| 型態 | 說明 | 範例 |
|------|------|------|
| 數值 Number | 可帶或不帶單位 | `12`、`100px` |
| 字串 String | 可帶或不帶引號 | `'Noto Sans TC'`、`bold` |
| 顏色 Color | 各種顏色表示法 | `blue`、`#4cb5fc`、`hsl(204, 97%, 64%)` |
| 列表 List | 空格或逗號分隔 | `1px 2px 3px`、`1px, 2px, 3px` |
| 地圖 Map | 鍵值對 | `('primary': blue, 'danger': red)` |
| 布林 Boolean | 真／假 | `true`、`false` |
| 空值 Null | 空 | `null` |

```scss
$font-size-h1: 50px;              // 數值
$font-color:   rgb(255, 81, 0);   // 顏色
$margin:       10px 20px 30px;    // list
$is-open:      true;              // 布林
$empty:        null;              // 空值
$bg-color: (                      // map
  'blue':   #0059ff,
  'yellow': #ffd900,
  'green':  #73ff00,
);
```

> Null 常用在 mixin 參數的預設值：某屬性傳 `null` 時，Sass 會**直接不輸出**該行 CSS（見第 6 章）。

### 變數作用域與 `!default`、`!global`

```scss
$color: red;          // 全域變數

.box {
  $color: blue;       // 區域變數（只在這個區塊有效）
  color: $color;      // blue
}

.text {
  color: $color;      // red（外層的全域值）
}
```

- `!default`：**只有在該變數尚未被定義時才賦值**，常用於可覆寫的設定檔。

```scss
$primary: #3b82f6 !default;  // 若使用者已定義 $primary 就不覆蓋
```

- `!global`：在區域中強制修改全域變數（少用）。

### Sass 變數 vs CSS 自訂屬性（CSS Variables）

| | Sass 變數 `$x` | CSS 變數 `--x` |
|---|---|---|
| 生效時機 | **編譯時**（編譯後就消失）| **執行時**（瀏覽器讀）|
| 能被 JS 讀寫 | ✗ | ✓ |
| 響應 media query / 動態切換 | ✗ | ✓ |

兩者可以搭配使用：用 Sass 管理設計 token，用 CSS 變數處理需要執行時切換的東西（例如深色模式）。

```scss
:root {
  --primary: #{$primary};  // 用 #{} 插值把 Sass 值塞進 CSS 變數
}
```

> `#{ }` 稱為 **插值（Interpolation）**，可把 Sass 的值插入到選擇器、屬性名、字串中。

只要出現在 **url 路徑／class 名稱／屬性名稱** 這三種位置，就一定要用 `#{}` 把變數包起來，否則變數不會被展開：

```scss
// 1) class 名稱
.#{$name} { width: 10px; }

// 2) url 路徑
.box { background-image: url(./images/#{$link}.jpg); }

// 3) 屬性名稱
.attr { #{$prop}: 10px; }
```

## 4. 巢狀 Nesting 與 `&` 父選擇器

巢狀讓你依照 HTML 結構撰寫，減少重複。

```scss
.card {
  padding: 16px;
  background: white;

  .title {
    font-size: 20px;
    font-weight: bold;
  }

  .content {
    color: #666;
  }
}
```

編譯後：

```css
.card { padding: 16px; background: white; }
.card .title { font-size: 20px; font-weight: bold; }
.card .content { color: #666; }
```

### `&` 父選擇器參照

`&` 代表「目前的父選擇器」，最常用來寫 **狀態（hover/focus）** 和 **BEM 命名**。

```scss
.button {
  background: $primary;

  &:hover {           // → .button:hover
    background: darken($primary, 10%);
  }

  &:focus { outline: 2px solid; }

  &--primary {        // → .button--primary（BEM 修飾子）
    background: green;
  }

  &.is-active {       // → .button.is-active
    font-weight: bold;
  }
}
```

`&` 也可以放在後面，表達「被某個父層包住時」：

```scss
.title {
  color: black;

  .dark-mode & {      // → .dark-mode .title
    color: white;
  }
}
```

### 巢狀屬性

同前綴的屬性也能巢狀（較少用）：

```scss
.box {
  font: {
    family: sans-serif;
    size: 16px;
    weight: bold;
  }
}
```

> **巢狀不要超過 3 層**！過深的巢狀會產生冗長、難覆寫、效能差的選擇器，是常見的壞味道。

## 5. Partials 與模組化 `@use` / `@forward`

### Partials

以 **底線開頭** 命名的檔案（如 `_variables.scss`）稱為 partial，它**不會被單獨編譯成 CSS**，只用來被其他檔案引入。

```
scss/
├── _variables.scss   # 變數
├── _mixins.scss      # mixin
├── _buttons.scss     # 按鈕元件
└── main.scss         # 主檔，把上面組合起來
```

### `@use`（現代做法，取代舊的 `@import`）

```scss
// main.scss
@use 'variables';
@use 'mixins';
@use 'buttons';
```

`@use` 引入時會有 **命名空間**，預設是檔名：

```scss
// _variables.scss
$primary: #3b82f6;

// main.scss
@use 'variables';

.button {
  color: variables.$primary;   // 用「命名空間.變數」存取
}
```

可以用 `as` 改別名，或 `as *` 拿掉命名空間：

```scss
@use 'variables' as v;
@use 'variables' as *;   // 之後可直接寫 $primary

.button { color: v.$primary; }
```

### 命名空間的關鍵規則：只看檔名，不看路徑

`@use` 的預設命名空間**只取檔名（去掉底線與副檔名），會忽略前面的資料夾路徑**：

```scss
@use 'components/button';   // 命名空間預設是 button，不是 components/button
```

這代表**兩個不同資料夾、但檔名相同的 partial**，只要都用預設命名空間 `@use`，就會撞名：

```scss
@use 'layout/button';       // 命名空間 → button
@use 'component/button';    // 命名空間也是 button → 撞名！

.x { color: button.$color; }
```

編譯會直接報錯：

```
Error: There's already a module with namespace "button".
```

解法是用 `as` 給每個模組一個好記、不重複的別名，一看就知道模組來源：

```scss
@use 'layout/button' as layout-btn;
@use 'component/button' as component-btn;

.x { color: layout-btn.$color; }
.y { color: component-btn.$color; }
```

> **實務建議**：資料夾式架構（像第 13 章示範的 `base/`、`component/`、`layout/`）特別容易踩到這個地雷，尤其是不同資料夾常常都有 `_index.scss`、`_button.scss` 這種通用命名。養成習慣：只要用 `@use` 引入巢狀路徑的檔案，就直接用 `as` 給明確別名，不要依賴預設命名空間。

### 用 `@use` 載入內建模組（`sass:*`）

`@use` 不只能載入你自己寫的 partial，也能載入 Sass **內建模組**，寫法完全一樣，只是路徑換成 `sass:模組名`：

```scss
@use "sass:map";

$theme: (
  'primary': #3b82f6,
  'radius': 8px,
);

.button {
  background: map.get($theme, 'primary');   // 用「命名空間.函式()」呼叫
  border-radius: map.get($theme, 'radius');
}
```

內建模組的命名空間預設就是**去掉 `sass:` 前綴後的模組名**（`sass:map` → `map`），一樣可以用 `as` 改別名：

```scss
@use "sass:math" as m;

.col {
  width: m.percentage(m.div(1, 3));
}
```

常用的內建模組：

| 模組 | 用途 | 常用函式 |
|------|------|----------|
| `sass:math` | 數學運算 | `math.div()`、`math.round()`、`math.percentage()` |
| `sass:color` | 顏色處理 | `color.adjust()`、`color.scale()`、`color.mix()` |
| `sass:map` | Map 操作 | `map.get()`、`map.set()`、`map.merge()`、`map.keys()` |
| `sass:list` | List 操作 | `list.nth()`、`list.length()`、`list.append()` |
| `sass:string` | 字串處理 | `string.length()`、`string.slice()`、`string.to-upper-case()` |
| `sass:selector` | 選擇器操作 | `selector.nest()`、`selector.append()` |
| `sass:meta` | 型別檢查、進階操作 | `meta.type-of()`、`meta.call()` |

> 每支檔案要用哪個內建模組的函式，就在檔案最上面 `@use` 該模組，跟載入自己的 partial 是同一套規則。第 8 章會示範更多內建函式的實際用法。

### `@forward`：轉發（做給別人用的樣式庫）

`@forward` 讓你把多個 partial 集中成一個入口再對外提供。

```scss
// _index.scss（樣式庫入口）
@forward 'variables';
@forward 'mixins';

// 使用端
@use 'index' as lib;
```

###關於舊的 `@import`

老教材常見 `@import 'variables';`。**Sass 官方已宣布逐步淘汰 `@import`**，因為它會造成全域命名衝突、重複載入。**新專案請一律使用 `@use` / `@forward`**。

## 6. Mixin：`@mixin` / `@include`

Mixin 是「可重用的樣式片段」，可以帶參數，是 Sass 最強大的功能之一。

### 基本用法

```scss
@mixin flex-center {
  display: flex;
  justify-content: center;
  align-items: center;
}

.box {
  @include flex-center;   // 引入
}
```

### 帶參數

```scss
@mixin button($bg, $color: white) {   // $color 有預設值
  background: $bg;
  color: $color;
  padding: 8px 16px;
  border-radius: 6px;
}

.btn-primary { @include button(#3b82f6); }
.btn-danger  { @include button(#ef4444, white); }
.btn-warn    { @include button($bg: #f59e0b); }  // 具名參數
```

### 可變參數與 `@content`

```scss
// 響應式斷點 mixin（超實用！）
@mixin respond($breakpoint) {
  @if $breakpoint == mobile {
    @media (max-width: 767px) { @content; }
  } @else if $breakpoint == tablet {
    @media (max-width: 1023px) { @content; }
  }
}

.container {
  width: 1200px;

  @include respond(mobile) {   // @content 會把這個區塊塞進 media query
    width: 100%;
  }
}
```

編譯後：

```css
.container { width: 1200px; }
@media (max-width: 767px) {
  .container { width: 100%; }
}
```

> **Mixin vs 函式**：mixin 產生「一段 CSS 規則」；函式（下面第 8 章）回傳「一個值」。

## 7. 繼承 `@extend` 與 `%placeholder`

`@extend` 讓一個選擇器繼承另一個選擇器的樣式，做法是**共用選擇器**（而非複製屬性）。

```scss
%card-base {          // % 開頭是「佔位選擇器」，本身不會輸出 CSS
  padding: 16px;
  border-radius: 8px;
  box-shadow: 0 2px 8px rgba(0,0,0,.1);
}

.card       { @extend %card-base; background: white; }
.card-dark  { @extend %card-base; background: #333; }
```

編譯後（注意選擇器被合併，樣式不重複）：

```css
.card, .card-dark {
  padding: 16px;
  border-radius: 8px;
  box-shadow: 0 2px 8px rgba(0,0,0,.1);
}
.card { background: white; }
.card-dark { background: #333; }
```

### `@extend` vs `@mixin` 該用哪個？

| | `@extend` / `%placeholder` | `@mixin` |
|---|---|---|
| 產出 | 合併選擇器，CSS 較精簡 | 每次複製一份樣式 |
| 能帶參數 | ✗ | ✓ |
| 能放 media query 內 | 有限制，容易出錯 | ✓ |
| 建議 | 純共用、無參數時 | **大多數情況優先用 mixin** |

> 實務建議：**不確定就用 mixin**。`@extend` 在大型專案容易產生非預期的選擇器爆炸。

## 8. 函式 `@function` 與內建函式

### 自訂函式

函式用 `@function` 定義、`@return` 回傳一個值。

```scss
@function rem($px, $base: 16px) {
  @return math.div($px, $base) * 1rem;
}

.title {
  font-size: rem(24px);   // → 1.5rem
}
```

> 注意：新版 Sass 除法要用 `math.div()`，需先 `@use 'sass:math';`。

### 常用內建模組與函式

新版 Sass 內建函式分在不同模組，需先 `@use`（載入語法見第 5 章「用 `@use` 載入內建模組」）：

```scss
@use 'sass:math';
@use 'sass:color';
@use 'sass:string';
@use 'sass:list';
@use 'sass:map';
```

**顏色相關（最常用）**：

```scss
color.adjust($primary, $lightness: -10%)   // 調整明度
color.scale($primary, $lightness: 20%)     // 按比例調整
color.mix(red, blue, 50%)                  // 混色
rgba($primary, 0.5)                         // 加透明度
```

> 舊寫法 `darken()` / `lighten()` / `mix()` 仍可用，但官方推薦改用 `color.adjust` / `color.scale`。

**數學相關**：

```scss
math.div(100, 3)     // 除法
math.round(4.6)      // 5
math.ceil(4.1)       // 5
math.percentage(0.5) // 50%
math.max(1, 5, 3)    // 5
```

## 9. 運算 Operators

Sass 支援數學運算，方便算佈局尺寸。

```scss
$gap: 16px;

.col {
  width: math.div(100%, 3);          // 三欄等寬
  padding: $gap * 2;                 // 32px
  margin-bottom: $gap / 2;           // 需用 math.div，見下
}
```

| 運算子 | 說明 |
|--------|------|
| `+ - * /` | 加減乘除 |
| `%` | 取餘數 |
| `== !=` | 相等 / 不相等 |
| `< > <= >=` | 比較 |
| `and or not` | 邏輯 |

> **除法陷阱**：`/` 在 CSS 有特殊意義（如 `font: 16px/1.5`），Sass 新版已**不建議**用 `/` 做除法，請改用 `math.div(a, b)`。

字串也能用 `+` 串接：

```scss
$name: 'icon';
.#{$name}-home { content: 'home'; }   // 搭配插值產生 .icon-home
```

## 10. 控制流程 `@if` / `@each` / `@for` / `@while`

Sass 有像程式語言一樣的流程控制，常用來**自動產生大量重複樣式**。

### `@if` / `@else`

```scss
@use 'sass:color';

@mixin text-color($bg) {
  // 取得 HSL 的明度通道（新版 Dart Sass 寫法）
  @if color.channel($bg, 'lightness', $space: hsl) > 50% {
    color: black;      // 淺底配黑字
  } @else {
    color: white;      // 深底配白字
  }
}
```

### `@each`：遍歷 List / Map

```scss
// 遍歷 List
$sizes: sm, md, lg;
@each $size in $sizes {
  .icon-#{$size} { /* ... */ }
}

// 遍歷 Map（產生一整組顏色工具類）
$colors: (
  'primary': #3b82f6,
  'success': #22c55e,
  'danger':  #ef4444,
);

@each $name, $color in $colors {
  .text-#{$name} { color: $color; }
  .bg-#{$name}   { background: $color; }
}
```

編譯後會自動產生 `.text-primary`、`.bg-primary`、`.text-success`… 一整組。

#### 實用範例：跨瀏覽器前綴

用 `@each` 遍歷各家瀏覽器前綴，一次補齊 `-webkit-`、`-moz-` 等：

```scss
@mixin prefix($property, $value) {
  @each $prefix in '', -webkit-, -moz-, -ms-, -o- {
    #{$prefix}#{$property}: $value;
  }
}

.box {
  @include prefix(transform, translateX(10px));
}
```

#### 實用範例：斷點 map + respond mixin

把斷點集中在一份 map，再用 `@each` 搭配 `@if` 做出響應式 mixin：

```scss
$breakpoints: (
  small:   480px,
  medium:  720px,
  desktop: 988px,
  large:   1366px,
);

@mixin respond($name) {
  @each $viewpoint, $width in $breakpoints {
    @if $name == $viewpoint {
      @media all and (min-width: $width) { @content; }
    }
  }
}

.container {
  @include respond(medium) {
    width: 100%;
  }
}
```

### `@for`：計數迴圈

```scss
// 產生 12 欄網格
@for $i from 1 through 12 {
  .col-#{$i} {
    width: math.percentage(math.div($i, 12));
  }
}
```

- `from 1 through 12`：包含 12（1~12）
- `from 1 to 12`：不含 12（1~11）

### `@while`

```scss
$i: 6;
@while $i > 0 {
  .mt-#{$i} { margin-top: #{$i * 4}px; }
  $i: $i - 1;
}
```

## 11. Maps 與 Lists

### Lists（清單）

以空格或逗號分隔的值集合。

```scss
$fonts: 'Noto Sans TC', system-ui, sans-serif;
$sizes: 8px 16px 24px 32px;

list.nth($sizes, 2)      // 取第 2 個 → 16px（索引從 1 開始）
list.length($sizes)      // 4
```

### Maps（鍵值對，超重要）

Map 是管理**設計系統 token** 的核心工具。

```scss
@use 'sass:map';

$theme: (
  'primary':   #3b82f6,
  'secondary': #64748b,
  'radius':    8px,
);

.button {
  background: map.get($theme, 'primary');   // 取值
  border-radius: map.get($theme, 'radius');
}

map.has-key($theme, 'primary')   // true
map.keys($theme)                 // primary, secondary, radius
```

搭配 `@each` 就能從一份 map 自動生成整套樣式，是 Sass 模組化的精髓。

## 12. 實戰整合：元件化的思維

把前面所有概念組起來，做一個可維護的按鈕系統：

```scss
@use 'sass:map';
@use 'sass:color';

// 1) 設計 token（Map）
$buttons: (
  'primary': #3b82f6,
  'success': #22c55e,
  'danger':  #ef4444,
);

// 2) 共用基礎（mixin）
@mixin button-base {
  display: inline-flex;
  align-items: center;
  padding: 8px 16px;
  border: none;
  border-radius: 6px;
  cursor: pointer;
  font-weight: 600;
  transition: background .2s;
}

// 3) 用迴圈生成各種顏色變體
.btn {
  @include button-base;

  @each $name, $color in $buttons {
    &--#{$name} {
      background: $color;
      color: white;

      &:hover {
        background: color.adjust($color, $lightness: -8%);
      }
    }
  }
}
```

這段會產生 `.btn--primary`、`.btn--success`、`.btn--danger`，每個都有對應 hover。**改 map 就能一次調整整套設計**——這就是 Sass 的價值。

## 13. `@import` 與傳統 Sass 專案架構

> 補充教材，架構參考自實際專案 [sexfat/sassProject](https://github.com/sexfat/sassProject)：一個用資料夾拆分、以 `@import` 組起來的傳統 Sass 專案。

### 為什麼還要學 `@import`？

官方建議新專案改用 `@use` / `@forward`（見第 5 章），但**舊專案、教學範例、許多既有的靜態網站**仍大量使用 `@import` 把整個網站的 SCSS 拆成資料夾管理。看得懂這種架構，才有辦法維護或延伸別人寫好的舊專案。

### 基本語法

```scss
@import 'var';                // 載入 _var.scss
@import 'base/setting';       // 載入 base/_setting.scss（可跨資料夾）
@import 'a', 'b', 'c';        // 一行載入多個檔案，逗號分隔
```

- 檔名一樣可以用底線開頭的 partial（`_var.scss`），但 `@import` 時**不用寫底線、不用寫副檔名**。
- 跟 `@use` 最大差異：`@import` 引入的變數 / mixin / 函式**直接進入全域作用域**，取用時不需要命名空間前綴。

```scss
// _var.scss
$primary: #3b82f6;

// style.scss
@import 'var';

.btn {
  background: $primary;   // 直接用，不用像 @use 那樣寫 var.$primary
}
```

### 實務案例：資料夾式架構

一個典型用 `@import` 組起來的專案，會依「用途」拆資料夾，最後在一支 `style.scss` 依序 `@import` 進來：

```
sass/
├── style.scss          # 入口，依序 @import 所有檔案
├── _var.scss            # 全域變數（顏色、間距…）
├── base/
│   ├── _setting.scss    # 全域設定、斷點
│   └── _color.scss      # 色彩系統
├── mixin/
│   └── _mixin.scss      # 共用 mixin
├── component/
│   └── _btn.scss        # 按鈕元件
└── layout/
    ├── _header.scss     # 頁首
    └── _footer.scss     # 頁尾
```

`style.scss` 的寫法：

```scss
// 共用設定（變數/設定要先載入，後面的檔案才能用）
@import 'var';
@import 'base/setting';
@import 'base/color';

// 複用樣式 mixin
@import 'mixin/mixin';

// 組件 component
@import 'component/btn';

// layout
@import 'layout/header';
@import 'layout/footer';
```

> **順序很重要**：`@import` 概念上像「複製貼上」，變數 / mixin 要先被定義，後面的檔案才用得到。常見順序是「共用設定 → mixin → component → layout → page」：先準備好工具，再組裝畫面。

完整可編譯範例在 [`examples/07-import-architecture/`](examples/07-import-architecture)：

```bash
sass sass/examples/07-import-architecture/style.scss out.css
```

### `@import` 的缺點

| 問題 | 說明 |
|------|------|
| 全域命名衝突 | 所有檔案的變數/mixin 都在同一個全域空間，容易撞名 |
| 重複載入 | 同一個檔案被多處 `@import`，內容會被重複編譯進去 |
| 難以追蹤來源 | 看到 `$primary` 不知道是哪個檔案定義的 |
| 官方逐步淘汰 | Dart Sass 已將 `@import` 標記為 deprecated，未來版本會移除 |

### 從 `@import` 遷移到 `@use`：用命名空間取代全域變數

新專案請直接用 `@use`（第 5 章）；維護舊專案時可以**照資料夾局部遷移**，重點是幫每個資料夾規劃一個清楚的命名空間：

```scss
// 每個資料夾內用 @forward 建一個入口
// base/_index.scss
@forward 'setting';
@forward 'color';

// component/_index.scss
@forward 'btn';

// layout/_index.scss
@forward 'header';
@forward 'footer';

// style.scss：主檔用 @use 逐一引入，並給每個模組一個好記的命名空間
@use 'var';                    // 命名空間：var
@use 'base';                   // 命名空間：base（只執行，不需要存取成員）
@use 'mixin/mixin' as m;       // 自訂命名空間：m
@use 'component' as c;         // 自訂命名空間：c（component/_btn.scss 已透過 @forward 併入）
@use 'layout' as l;            // 自訂命名空間：l

// 需要另外用到變數/mixin 時，透過命名空間存取，來源一目了然
.page-hero {
  @include m.flex-center;
  background: var.$primary;    // 一看就知道 $primary 來自 var 模組
}
```

跟 `@import` 版本比起來，差別在於：**每個變數、mixin 都清楚標示來自哪個命名空間**，不再是「全域一大包」，也不用擔心第 5 章提到的 `button` 撞名問題——就算 `component/_btn.scss` 跟未來新增的 `page/_btn.scss` 同名，只要各自 `@forward` 進不同資料夾的 `_index.scss`，最外層用 `component`／`page` 這種命名空間區分，就不會衝突。

## 重點速記

- 用 `$` 定義變數集中管理設計值。
- 巢狀不超過 3 層，善用 `&` 寫狀態與 BEM。
- **模組化新專案用 `@use` / `@forward`，別再用 `@import`**；維護舊專案時才需要看懂資料夾式 `@import` 架構（第 13 章）。
- 重用邏輯優先用 **mixin**（可帶參數）；純共用無參數才考慮 `@extend`。
- 除法用 `math.div()`，顏色用 `color.adjust()`。
- Map + `@each` = 自動生成設計系統。
