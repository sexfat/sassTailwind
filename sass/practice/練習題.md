# Sass 練習題

> 每題請自己先寫，再對照 `解答/` 資料夾。建議邊寫邊用 `sass 檔名.scss out.css` 編譯，觀察輸出的 CSS。

## 怎麼用 AI 幫你學（先看這裡）

每題下方都有「AI 協助」，附上可直接複製的 prompt。寫 prompt 的 4 個要領：

- **給情境**：開頭說明「我在學 Sass/SCSS，請用現代語法（`@use`、`math.div`）」，AI 才不會給過時寫法。
- **先要提示、別急著要答案**：加一句「先給我思路與提示，不要直接給完整程式碼」，練習效果最好。
- **卡住再貼程式碼除錯**：把你寫的 SCSS 貼上，問「為什麼編譯錯誤？」或「輸出的 CSS 為什麼不對？」。
- **要求解釋**：拿到答案後追問「請逐行解釋這段」「還有沒有更精簡的寫法？」。

## 基礎題

### 練習 1：變數與運算

> Figma 參考樣式：[開啟「Button」元件](https://www.figma.com/design/e87BnLqAVr9tTSyCTNP7TX/?node-id=5-27)

定義主色 `$primary: #6366f1`、基準間距 `$space: 8px`。做出 `.btn`：

- 背景用主色、文字白色
- 上下 padding 為 `$space`、左右為 `$space` 的 3 倍
- 圓角 `$space`

> **AI 協助 · 可以這樣問**
> ```
> 我在學 Sass。想用變數做一個 .btn 按鈕：
> 主色 $primary:#6366f1、基準間距 $space:8px。
> 需求：背景主色、白字、上下 padding 用 $space、左右用 $space 的 3 倍、圓角 $space。
> 請先給我思路與會用到的語法提示，「不要」直接給完整答案。
> ```
> 寫出來後可再問：「左右 padding 要怎麼用運算寫成 $space 的 3 倍？」

### 練習 2：巢狀與 `&`
寫一個 `.nav`，內含 `.nav__item`（巢狀），並用 `&` 完成：

- `.nav__item` hover 時文字變成 `$primary`
- `.nav__item` 加上 `--active` 修飾子時字重 bold（`.nav__item--active`）

> **AI 協助 · 可以這樣問**
> ```
> 我在學 Sass 的巢狀與 & 父選擇器。
> 請教我怎麼用 & 寫 BEM：一個 .nav 內含 .nav__item，
> 其中 hover 變色、加 --active 修飾子時字重 bold。
> 先解釋 & 在 &__item、&:hover、&--active 各自會編譯成什麼，再讓我自己寫。
> ```
> 追問：「為什麼 &__item 會變成 .nav__item 而不是 .nav .item？」能幫你真的理解 &。

### 練習 3：Mixin
寫一個 `@mixin flex($justify: center, $align: center)`，能設定 `display:flex` 與傳入的對齊方式。用它做出 `.header`（兩端對齊 `space-between`、垂直置中）。

> **AI 協助 · 可以這樣問**
> ```
> 請教我 Sass 的 @mixin 帶參數＋預設值。
> 我想寫 @mixin flex($justify: center, $align: center) 設定 flex 與對齊，
> 再用 .header 呼叫它做出 space-between + 垂直置中。
> 請先說明「參數預設值」和「具名參數呼叫」的語法，再讓我試寫。
> ```
> 進階：問「這個跟直接寫 display:flex 有什麼好處？什麼時候該用 mixin？」

## 進階題

### 練習 4：響應式 Mixin
寫一個 `@mixin respond($bp)`，支援 `mobile`（max-width 767px）與 `tablet`（max-width 1023px），使用 `@content`。用它讓 `.container` 在 mobile 時寬度變 100%。

> **AI 協助 · 可以這樣問**
> ```
> 我想用 Sass 做響應式 mixin。請教我 @content 的用法：
> 寫 @mixin respond($bp)，用 @if / @else if 判斷 mobile(max-width:767px) 與 tablet(max-width:1023px)，
> 把 @content 塞進對應的 @media 內。
> 先解釋 @content 是什麼、呼叫時的 @include respond(mobile){...} 大括號會跑去哪裡。
> ```
> 想更進階可問：「怎麼改用斷點 map + @each 讓斷點集中管理？」

### 練習 5：迴圈生成間距工具類
用 `@for` 產生 `.m-1` ~ `.m-5`（margin）與 `.p-1` ~ `.p-5`（padding），每一級是 `4px * 級數`。
（例如 `.m-3 { margin: 12px; }`）

> **AI 協助 · 可以這樣問**
> ```
> 請教我 Sass 的 @for 迴圈自動生成工具類。
> 我要產生 .m-1~.m-5 和 .p-1~.p-5，每級 margin/padding = 4px × 級數。
> 請說明 @for $i from 1 through 5 跟 to 的差別，以及怎麼用 #{$i} 把變數插進 class 名稱。
> 先給提示，我自己寫寫看。
> ```
> 卡在 class 名稱沒展開時，貼上你的程式碼問：「為什麼 class 變成 .m-$i 而不是 .m-1？」

### 練習 6：Map + `@each`

> Figma 參考樣式：[開啟「Alert」元件](https://www.figma.com/design/e87BnLqAVr9tTSyCTNP7TX/?node-id=6-31)

給定：

```scss
$status: (
  'info': #3b82f6,
  'success': #22c55e,
  'warning': #f59e0b,
  'error': #ef4444,
);
```

自動生成 `.alert--info`、`.alert--success`… 每個要有：對應背景色、白色文字、padding 12px、圓角 6px。

> **AI 協助 · 可以這樣問**
> ```
> 我有一個 Sass map $status（info/success/warning/error 對應顏色）。
> 請教我用 @each $name, $color in $status 自動生成 .alert--#{$name}，
> 每個要有對應背景色、白字、padding 12px、圓角 6px。
> 先解釋 @each 遍歷 map 時 $name 和 $color 分別拿到什麼。
> ```
> 追問：「這樣寫跟手動複製四份 .alert 相比，之後要新增一個顏色時差在哪？」

## 挑戰題

### 練習 7：迷你設計系統

> Figma 參考樣式：[開啟「Card」元件](https://www.figma.com/design/e87BnLqAVr9tTSyCTNP7TX/?node-id=6-20)

綜合運用**模組化 + Map + mixin + 迴圈**，做一個卡片元件系統：

1. 把變數放進 `_tokens.scss`（partial），用 `@use` 引入。
2. 用一個 Map 定義三種卡片主題色。
3. 寫 `@mixin card-base` 放共用樣式（padding、圓角、陰影、transition）。
4. 用 `@each` 生成 `.card--primary` / `.card--success` / `.card--danger`，各自有主題色的上邊框，hover 時陰影加深。

完成後編譯，檢查輸出的 CSS 是否精簡且正確。

> **AI 協助 · 可以這樣問（挑戰題建議分段問）**
> ```
> 我要用 Sass 做一個卡片設計系統，請「分步驟」帶我，一次只給一小段：
> 步驟1：把 $card-radius、$card-padding、$card-themes(map) 放進 _tokens.scss（partial），
>        主檔怎麼用 @use '_tokens' as t 引入並加命名空間？
> 步驟2：寫 @mixin card-base 放共用樣式（padding、圓角、陰影、transition）。
> 步驟3：用 @each 生成 .card--primary/success/danger，各自主題色上邊框、hover 陰影加深。
> 每一步請先解釋再給範例，讓我跟著做。
> ```
> 挑戰題別一次要全部答案。分步問，並在每步用 `sass 練習7.scss out.css` 編譯驗證，再進下一步。
