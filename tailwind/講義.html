Tailwind CSS 完整講義

Tailwind CSS 是一套 Utility-First（工具優先） 的 CSS 框架。

它不給你「元件」，而是給你一堆小的、單一用途的 class（例如 flex、pt-4、text-center），讓你直接在 HTML 裡「組合」出樣式，幾乎不用自己寫 CSS。

1. 什麼是 Utility-First？

傳統寫法：先想一個 class 名稱，再到 CSS 檔寫規則。

html
<div class="card">...</div>
css
.card {
  padding: 16px;
  border-radius: 8px;
  background: white;
  box-shadow: 0 2px 8px rgba(0,0,0,.1);
}

Tailwind 寫法：直接在 HTML 用工具類組合，不寫 CSS。

html
<div class="p-4 rounded-lg bg-white shadow-md">...</div>
  • p-4 = padding: 1rem
  • rounded-lg = border-radius: 0.5rem
  • bg-white = background: white
  • shadow-md = 一組預設陰影

為什麼這樣比較好？

  • 不用想 class 名字：不再糾結 .card-wrapper-inner 該叫什麼。
  • 不用切換檔案：改樣式就在 HTML 當下改，所見即所得。
  • 不會樣式衝突：沒有全域 CSS 互相覆蓋的問題。
  • 檔案不膨脹：Tailwind 只把「你用到的」class 打包進最終 CSS，沒用到的自動移除。
  • 設計一致性：間距、顏色、字級都來自預設的設計系統（4 8 16… 而非隨手 13px）。

常見疑慮：「HTML 會不會很醜很長？」

會比較長，但實務上：搭配元件框架（React/Vue）把重複的東西封裝成元件，或用 @apply 抽出常用組合即可。多數團隊實測後認為維護性利大於弊。

2. 與傳統 CSS / Bootstrap 的差異

傳統 CSS / SassBootstrapTailwind CSS
核心單位自訂 class現成元件（.btn、.card）工具類（.flex、.p-4）
客製彈性高（但要自己寫）低（改元件較麻煩）高（組合工具類）
長相是否雷同不會容易「一看就是 Bootstrap」不會
學習重點CSS 屬性記元件與 grid 系統記工具類命名規則
最終檔案大小看你怎麼寫較大（整包）很小（只留用到的）

一句話：Bootstrap 給你組好的家具，Tailwind 給你標準化的積木。

3. 安裝與環境（CDN / CLI / Vite）

本教材以 Tailwind CSS v4 為主（官方最新），並標註 v3 差異。

方法 A：Play CDN（練習／Demo 最快，不需安裝）

在 HTML <head> 加一行，即可使用所有工具類。僅適合學習與雛形，不建議上線（檔案較大、無法完整客製）。

html
<!-- Tailwind v4 瀏覽器版 -->
<script src="https://unpkg.com/@tailwindcss/browser@4"></script>

v3 的 Play CDN 是 <script src="https://cdn.tailwindcss.com"></script>。

本教材 examples/ 內的 HTML 都用這種方式，直接用瀏覽器打開就能看效果。

方法 B：Tailwind CLI

bash
npm install tailwindcss @tailwindcss/cli

# 建立輸入檔 input.css，內容只要：
#   @import "tailwindcss";

# 編譯 + 監看
npx @tailwindcss/cli -i input.css -o output.css --watch

方法 C：Vite（正式專案推薦）

bash
npm create vite@latest my-app
cd my-app
npm install tailwindcss @tailwindcss/vite

vite.config.js：

js
import tailwindcss from '@tailwindcss/vite';

export default {
  plugins: [tailwindcss()],
};

src/style.css：

css
@import "tailwindcss";

v3 差異：v3 需要 tailwind.config.js 設定 content 掃描路徑，並在 CSS 用 @tailwind base; @tailwind components; @tailwind utilities;。v4 大幅簡化：一行 @import "tailwindcss";，會自動偵測要掃描的檔案。

4. 核心工具類速查

工具類命名有固定規則，記住規則就能舉一反三。

間距（Spacing）— 最常用

數字是「間距刻度」，1 = 0.25rem = 4px，所以 p-4 = 16px。

Class意義
p-4 / px-4 / py-4 / pt-4padding（全 / 左右 / 上下 / 上）
m-4 / mx-auto / mt-2margin（mx-auto 水平置中）
gap-4flex/grid 的間距
space-x-4子元素之間的水平間距

尺寸（Sizing）

Class意義
w-full / w-1/2 / w-64寬 100% / 50% / 16rem
h-screen高度 = 視窗高
max-w-md最大寬度（sm/md/lg/xl…）
min-h-screen最小高度滿版

文字（Typography）

Class意義
text-sm / text-lg / text-2xl字級
font-bold / font-medium字重
text-center / text-right對齊
text-gray-600文字顏色（顏色-深淺）
leading-relaxed行高
tracking-wide字距

顏色（Color）

命名規則：屬性-顏色-深淺，深淺從 50（最淺）到 950（最深）。

html
<div class="bg-blue-500 text-white border-blue-700">...</div>
  • 背景：bg-red-500
  • 文字：text-green-600
  • 邊框：border-gray-300
  • 常見色系：slate gray red orange amber yellow green teal blue indigo purple pink

邊框與圓角、陰影

Class意義
border / border-2邊框粗細
rounded / rounded-lg / rounded-full圓角
shadow / shadow-md / shadow-xl陰影

顯示與定位

Class意義
block / inline-block / hiddendisplay
relative / absolute / fixedposition
top-0 / right-4 / inset-0定位偏移
z-10z-index
overflow-hidden / overflow-auto溢出處理

任意值（Arbitrary values）

預設刻度不夠用時，可用方括號寫任意值：

html
<div class="top-[117px] bg-[#1da1f2] w-[73%] text-[13px]">...</div>

5. Flexbox 與 Grid 佈局

Flexbox

html
<div class="flex items-center justify-between gap-4">
  <div>左</div>
  <div>右</div>
</div>
Class意義
flexdisplay: flex
flex-col改為直向排列
items-center交叉軸置中（align-items）
justify-between主軸兩端對齊（justify-content）
justify-center主軸置中
flex-wrap允許換行
flex-1佔滿剩餘空間

Grid

html
<div class="grid grid-cols-3 gap-4">
  <div>1</div><div>2</div><div>3</div>
</div>
Class意義
griddisplay: grid
grid-cols-33 欄
col-span-2跨 2 欄
gap-4格線間距

經典置中（水平垂直都置中）：

html
<div class="flex items-center justify-center h-screen">
  <div>置中內容</div>
</div>

6. 響應式設計 RWD（斷點）

Tailwind 是 Mobile First：沒加前綴的樣式是手機版，加斷點前綴才是「該尺寸(含)以上」套用。

前綴生效寬度對應裝置
（無）全部手機
sm:≥ 640px大手機 / 直立平板
md:≥ 768px平板
lg:≥ 1024px筆電
xl:≥ 1280px桌機
2xl:≥ 1536px大桌機

範例：手機一欄、平板兩欄、桌機四欄。

html
<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-4">
  ...
</div>

常見錯誤：想在「手機」套樣式卻寫 sm:。記住——手機版不用前綴，sm: 是 640px 以上。

7. 狀態變體：hover / focus / group / peer

用「變體前綴 :」在特定狀態套樣式。

html
<button class="bg-blue-500 hover:bg-blue-700 focus:ring-2 active:scale-95">
  按鈕
</button>

常用變體：hover: focus: active: disabled: focus-within: first: last: odd: even:

group：依「父層狀態」改變子元素

在父層加 group，子元素用 group-hover: 反應父層的 hover。

html
<a href="#" class="group block p-4 border">
  <h3 class="group-hover:text-blue-600">標題</h3>
  <p class="text-gray-500 group-hover:text-gray-700">滑到整張卡都會變色</p>
</a>

peer：依「兄弟元素狀態」改變

常用於表單：input 有 peer，後面的元素用 peer-focus:、peer-invalid: 反應。

html
<input type="email" class="peer border" />
<p class="hidden peer-invalid:block text-red-500">Email 格式錯誤</p>

8. 深色模式 Dark Mode

用 dark: 前綴指定深色模式下的樣式。

html
<div class="bg-white text-gray-900 dark:bg-gray-800 dark:text-white">
  自動或手動切換深淺色
</div>
  • 預設：跟隨作業系統設定（prefers-color-scheme）。
  • 手動切換：在 <html> 加/移除 class="dark"（需設定 selector 策略），用 JS 切換：
js
document.documentElement.classList.toggle('dark');

v4 設定手動策略：在 CSS 加 @custom-variant dark (&:where(.dark, .dark *));。

v3 則在 tailwind.config.js 設 darkMode: 'class'。

9. 自訂主題（v4 @theme 與 v3 config）

v4：用 CSS 的 @theme（推薦）

v4 把設定搬進 CSS，用 @theme 定義設計 token，會自動生成對應工具類。

css
@import "tailwindcss";

@theme {
  --color-brand: #6366f1;        /* 產生 bg-brand / text-brand… */
  --color-brand-dark: #4f46e5;
  --font-display: 'Noto Sans TC', sans-serif;
  --spacing-18: 4.5rem;          /* 產生 p-18 / m-18… */
  --breakpoint-3xl: 1920px;      /* 新增 3xl: 斷點 */
}
html
<button class="bg-brand hover:bg-brand-dark font-display p-18">品牌按鈕</button>

v3：用 tailwind.config.js

js
module.exports = {
  content: ['./src/**/*.{html,js}'],
  theme: {
    extend: {
      colors: {
        brand: { DEFAULT: '#6366f1', dark: '#4f46e5' },
      },
      spacing: { 18: '4.5rem' },
    },
  },
};

extend 是「新增」；直接寫在 theme 底下會「覆蓋」預設值，通常用 extend。

10. 重複樣式的處理：@apply 與元件抽取

當同一組工具類到處重複（例如按鈕），有兩種做法：

做法 A：元件化（首選）

用 React/Vue 把它包成元件，class 只寫一次：

jsx
function Button({ children }) {
  return <button className="px-4 py-2 rounded-lg bg-blue-500 text-white hover:bg-blue-600">{children}</button>;
}

做法 B：@apply 抽出自訂 class

在 CSS 用 @apply 把工具類組合成語意化 class：

css
@import "tailwindcss";

@layer components {
  .btn {
    @apply px-4 py-2 rounded-lg font-semibold transition;
  }
  .btn-primary {
    /* 直接列出 utility，不要用 @apply 去串接另一個自訂類 .btn */
    @apply px-4 py-2 rounded-lg font-semibold transition
           bg-blue-500 text-white hover:bg-blue-600;
  }
}
html
<button class="btn-primary">送出</button>

兩個提醒：

1. 別過度使用 @apply。官方建議 優先用元件抽取，@apply 只在少數真的重複、又無法元件化時使用（否則就退回傳統 CSS 的老問題了）。

2. @apply 請直接列 utility class，不要 @apply btn ... 去串接「另一個自訂類」——v4 中這種串接不保證有效，把 utility 直接寫出來最穩妥。

11. 推薦工具與最佳實踐

必裝工具

  • Tailwind CSS IntelliSense（VS Code 擴充）：class 自動補全、hover 顯示對應 CSS、顏色預覽。強烈建議必裝。
  • Prettier + prettier-plugin-tailwindcss：自動把 class 依官方順序排序，維持整潔。
  • 官方 Playground（play.tailwindcss.com）：線上即時試玩、看編譯結果。

最佳實踐

  1. 善用設計刻度：優先用 p-4、text-lg 這種刻度，少用 p-[13px] 任意值，維持一致性。
  2. Mobile First：先寫手機版（無前綴），再往上加 md:、lg:。
  3. 重複就抽元件，而不是狂複製 class。
  4. class 排序交給 Prettier，不用手動整理。
  5. 不確定 class 名稱：靠 IntelliSense 或查官方文件，不要硬背全部。

重點速記

  • Utility-First：在 HTML 組合小工具類，不寫 CSS。
  • 間距刻度 1 = 4px，p-4 = 16px。
  • 顏色是 屬性-色系-深淺（50~950）。
  • Mobile First：無前綴是手機，md:/lg: 是往上加。
  • 狀態用 hover:/focus:，父子連動用 group/peer。
  • 深色模式用 dark:。
  • v4 用 @import "tailwindcss" + @theme 客製；v3 用 config 檔。
  • 重複優先「元件化」，少用 @apply。
